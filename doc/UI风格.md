# PaaS Web Console UI 风格

## 参考图

参考根目录图片：

```text
xx.png
```

该图片定义 PaaS Web Console 的主要视觉方向。后续前端实现应以此为基准，而不是重新设计成营销页、大屏或装饰型后台。

## 风格定位

```text
企业级运维控制台
安静
清晰
信息密度适中偏高
适合长时间操作
强调状态、校验、流程和可扫描性
```

## 语言规范

- Web Console 所有用户可见文案必须使用中文，不使用英文作为页面标题、导航菜单、按钮、表单标签、提示、状态、空状态或错误文案。
- 技术枚举、API 字段、构建日志原文、代码片段和第三方产品名可以保留英文，例如 `java_springboot`、`build_command`、GitLab、Jenkins、Argo CD。
- 页面示例中的按钮和菜单文案均按中文实现。

```text
React
```

## 全局布局

布局结构：

```text
左侧深色导航栏
顶部全局工具栏
主内容区域
底部固定操作栏，仅用于向导或关键流程
```

左侧导航：

- 深色背景。
- 顶部显示产品名 `平台控制台`。
- 使用图标 + 文本。
- 当前菜单使用蓝色高亮背景和左侧强调线。
- 分组显示 `平台` 和 `平台管理`。
- 底部提供折叠入口。

顶部工具栏：

- 显示当前组织或租户上下文。
- 提供全局搜索框，支持搜索应用、资源、文档。
- 右侧提供帮助、通知、用户头像和用户菜单。

主内容区：

- 白色或极浅灰背景。
- 内容左对齐，使用较宽工作区。
- 标题区简洁，不使用 hero、插画或大面积渐变。
- 页面之间保持统一的页边距、卡片间距和表单密度。

## 创建应用向导布局

创建应用向导必须参考 `xx.png` 的结构。

页面结构：

```text
页面标题
横向步骤条
说明文案
三栏工作区
底部操作栏
```

步骤条：

- 横向展示。
- 当前步骤使用蓝色圆形数字。
- 已完成步骤使用绿色勾选。
- 未完成步骤使用灰色数字。
- 步骤之间使用浅色连接线。

三栏工作区：

```text
左栏：表单配置
中栏：仓库/模板/资源预览
右栏：校验清单
```

左栏表单：

- 使用 Ant Design Form。
- label 左对齐。
- 必填项使用红色星号。
- 每个字段下方显示简短辅助说明。
- 输入框、选择器高度和宽度统一。

中栏预览：

- 创建应用源码步骤显示仓库树。
- 部署模板步骤显示模板文件树或模板预览。
- 选中节点使用浅蓝背景。
- 底部提供外部查看链接，但不暴露用户不该直接访问的系统入口。

右栏校验清单：

- 使用卡片列表。
- 成功项使用绿色图标。
- 警告项使用橙色图标。
- 失败项使用红色图标。
- 每项可展开查看说明。
- 所有阻断项通过后才能进入下一步。

底部操作栏：

- 左侧 `返回`。
- 右侧 `校验` 和 `下一步`。
- 主按钮使用蓝色。
- 操作栏固定在页面底部或向导容器底部。

## 登录页

- 登录页提供本地账号密码登录表单，字段文案使用 `账号`、`密码`、`登录`。
- 登录页提供 OIDC 登录入口，按钮文案使用 `使用企业身份登录` 或具体身份源中文名称。
- 登录失败、账号禁用、OIDC 回调失败、会话过期等错误提示必须使用中文。

## 页面组件风格

应用列表：

- 表格为主。
- 支持搜索、项目筛选、环境状态筛选。
- 环境状态用 tag、badge 或 compact status group 展示。

应用详情：

- 顶部摘要区。
- 下方使用 Tabs：总览、环境、版本、构建、发布、配置、日志、监控、设置。
- 不使用嵌套卡片堆叠，卡片只用于独立信息块。

环境详情：

- 展示当前版本、期望版本、集群绑定、实例状态、同步状态、健康状态。
- 事件流使用 Timeline 或表格。

构建详情：

- 上方展示构建元信息。
- 日志区域使用终端风格。
- 支持自动滚动、暂停滚动、搜索、只看错误。

发布晋级：

- 展示版本、镜像 digest、commit、构建号、目标环境、审批策略。
- 生产发布明确展示审批人数、审批人范围、禁止自审批和审批进度。

部署模板配置：

- 左侧或中间使用 Monaco Editor。
- 右侧显示校验结果和版本记录。
- 支持 initContainer、目录创建、目录权限、volumeMount、安全上下文等模板字段编辑。
- 模板变更必须显示校验状态和审计提示。

## 颜色和状态

主色：

```text
primary = blue
success = green
warning = orange
error = red
neutral = gray
sidebar = dark navy
```

状态使用：

- 成功、健康、可用：绿色。
- 运行中、同步中、当前步骤：蓝色。
- 待审批、警告、非阻断问题：橙色。
- 失败、异常、阻断校验：红色。
- 禁用、未知、未开始：灰色。

## 禁止事项

- 不做营销落地页。
- 不做大屏风格。
- 不使用大面积渐变、装饰图、玻璃拟态或过度动效。
- 不把页面做成卡片套卡片。
- 不在首屏展示与操作无关的说明性大段文字。
- 不让用户直接跳转到 Jenkins UI、Argo CD UI、部署清单仓库或 kubeconfig。

## 验收标准

- Web Console 首屏应呈现与 `xx.png` 一致的企业控制台风格。
- 创建应用向导应实现左表单、中预览、右校验清单三栏布局。
- 页面元素在桌面端和常见笔记本分辨率下不重叠、不溢出。
- Ant Design 组件主题应统一，导航、按钮、状态色、表单密度一致。
- 核心流程页面可以通过 mock API 独立展示和测试。
