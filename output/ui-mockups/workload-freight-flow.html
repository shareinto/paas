设计规则

✓
一个镜像一个 Workload
每个 Workload 独立扩缩容、探针、配置、网络访问和回滚。
✓
Freight 是完整应用版本
创建时必须覆盖全部启用 Workload。
!
允许自定义镜像
允许 tag 或 digest，使用 tag 时提示回滚不可完全复现。
✓
环境差异不放入 Freight
dev/prod 的 values、域名、资源和环境变量归属 Stage/Environment。

创建 Freight

系统会自动列出全部启用 Workload。每个 Workload 必须选择镜像版本，不能少选、不能重复、不能删除行。
