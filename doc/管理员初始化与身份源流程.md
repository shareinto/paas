# 管理员初始化、OIDC Provider 配置和 Agent token 轮换流程

本文档记录测试可用版管理员初始化、身份源配置和集群 Agent token 轮换流程。所有操作必须通过 PaaS 控制面 API 或控制台完成，不向普通用户暴露 Jenkins、Argo CD、部署清单仓库或 kubeconfig。

## 管理员初始化

当前开发版 `paas-server` 启动时会写入一个演示管理员账号：

```text
username = admin
password = password
```

该账号仅用于本地开发和测试可用版本演示。测试环境和试点环境应在首次启动后立即通过 `POST /api/users` 创建正式管理员或运维用户，并通过 `POST /api/users/{userId}/reset-password` 重置演示账号密码或禁用演示账号。

创建本地用户示例：

```bash
curl -fsS -X POST "$PAAS_URL/api/users" \
  -H 'Content-Type: application/json' \
  -d '{
    "actor_id": "usr_admin",
    "username": "ops-admin",
    "password": "replace-with-initial-password",
    "display_name": "平台管理员",
    "email": "ops-admin@example.local"
  }'
```

密码只在请求中出现一次，后端保存 bcrypt 哈希，不在 API 响应中返回。

## OIDC Provider 配置

创建 OIDC Provider：

```bash
curl -fsS -X POST "$PAAS_URL/api/auth/oidc/providers" \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "企业身份",
    "issuer": "https://idp.example.local",
    "client_id": "paas-console",
    "client_secret_ref": "secret/oidc/paas-console",
    "scopes": ["openid", "profile", "email"],
    "redirect_uri": "https://paas.example.local/oidc/callback",
    "enabled": true
  }'
```

查询可用 OIDC Provider：

```bash
curl -fsS "$PAAS_URL/api/auth/oidc/providers"
```

响应只返回 `id`、`name`、`issuer`、`client_id`、`scopes`、`redirect_uri` 和 `enabled`，不会返回 `client_secret_ref` 或 OIDC client secret 明文。

发起登录：

```bash
curl -fsS "$PAAS_URL/api/auth/oidc/<providerId>/start"
```

## Agent token 生成和轮换

注册集群时控制面返回一次性 Agent token：

```bash
curl -fsS -X POST "$PAAS_URL/api/clusters" \
  -H 'Content-Type: application/json' \
  -d '{
    "tenant_id": "tenant_1",
    "name": "开发集群",
    "region": "cn"
  }'
```

响应中的 `agent_token` 只展示一次，应立即写入部署环境的 Kubernetes Secret：

```bash
kubectl -n paas-system create secret generic paas-agent-secret \
  --from-literal=PAAS_AGENT_TOKEN='<agent_token>'
```

轮换 Agent token：

```bash
curl -fsS -X POST "$PAAS_URL/api/clusters/<clusterId>/rotate-token"
```

轮换响应同样只返回一次性 `agent_token`，不会返回 `agent_token_hash`。轮换后旧 token 立即失效，需要更新 `paas-agent` Secret 并滚动重启 Agent。

## 安全要求

- 本地用户密码、OIDC client secret、Agent token 不写入文档、日志或返回值。
- Agent token 只能上报所属 `clusterId`，跨集群上报会被拒绝。
- OIDC Provider 响应不包含 `client_secret_ref`。
- 集群列表和查询响应不包含 `agent_token_hash`。
- token 轮换、用户创建、密码重置和登录应纳入审计或运行日志检查。
