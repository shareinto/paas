# 测试可用版本端到端验收清单

本文档用于将测试可用版本从“开发完成”推进到“可验收、可部署、可内部试点”。验收以 PaaS 控制台和 PaaS API 为唯一用户入口，不允许普通用户直接访问 Jenkins、Argo CD、部署清单仓库或 kubeconfig。

## 1. 验收准备

- [ ] 准备 MySQL 8.0+，存储引擎使用 InnoDB，字符集使用 utf8mb4。
- [ ] 准备企业内部 GitLab，包含源码仓库 Group 和部署清单仓库 Group。
- [ ] 准备 Jenkins，并配置平台统一构建 Job 模板。
- [ ] 准备镜像仓库，并配置 Spring Boot 和 Tomcat 运行时基础镜像。
- [ ] 准备至少一个 Kubernetes 集群，集群内已安装 Argo CD。
- [ ] 在每个 Kubernetes 集群内通过 `deploy/paas-agent` Helm Chart 部署 PaaS Agent 和最小 RBAC。
- [ ] 配置 OIDC Provider，同时保留本地用户账号密码登录能力。
- [ ] 初始化平台管理员、本地测试用户、租户、项目和必要 RBAC 权限。
- [ ] 配置 `paas-server` 到 MySQL、GitLab、Jenkins、OIDC 和镜像仓库的连接。
- [ ] 配置 `paas-agent` 到 PaaS 控制面的地址和 Agent token。

## 2. 主链路验收

- [ ] 用户可以使用本地账号密码登录 PaaS 控制台。
- [ ] 用户可以使用 OIDC 登录 PaaS 控制台。
- [ ] 平台管理员可以创建租户、项目和项目成员。
- [ ] 用户可以在流水线中直接填写 Git 或 SVN 源码地址。
- [ ] Git 源码地址可以刷新 GitLab 分支列表并选择构建分支。
- [ ] GitLab 成员权限按 PaaS 角色同步，且只表示源码权限。
- [ ] 用户创建 Application 不需要先选择 SourceRepository。
- [ ] 创建 Application 时可以指定 `source_path` 和 BuildSpec。
- [ ] 用户可以在 Application 下创建至少一个 Workload。
- [ ] Workload 支持选择 Deployment 或 StatefulSet。
- [ ] Workload 支持绑定流水线产物来源或输入自定义镜像来源。
- [ ] Workload 部署配置支持端口、资源、探针、域名、配置文件和可写目录。
- [ ] BuildSpec 允许 `java_springboot` 和 `java_tomcat`。
- [ ] 创建 Application 后可立即感知 `dev`、`test`、`staging`、`prod` Stage 投影。
- [ ] 无可用集群绑定时，Stage 进入 `pending_cluster_binding`，构建允许执行，部署被阻止。
- [ ] 有可用集群绑定时，PaaS 创建部署清单路径和 Argo CD Application 清单。
- [ ] 用户可以在 PaaS 中触发 Jenkins 构建。
- [ ] Jenkins Job 在点击构建时创建或更新，名称符合 `paas/{tenant_name}/{project_name}/{app_name}`。
- [ ] 构建参数来自 ApplicationSource 中固化的 BuildSpec。
- [ ] 用户可以在 PaaS 中实时查看构建日志。
- [ ] 构建成功后生成 BuildArtifact 和对应 Workload 的 Release 候选，不自动生成 Freight。
- [ ] 用户可以手动创建 Freight。
- [ ] 创建 Freight 时必须列出 Application 下所有启用 Workload。
- [ ] 创建 Freight 时每个启用 Workload 必须且只能选择一个镜像版本。
- [ ] FreightItem 支持从流水线产物选择镜像。
- [ ] FreightItem 支持用户输入自定义镜像。
- [ ] 自定义镜像使用 tag 时，Web Console 展示 tag 漂移风险提示。
- [ ] 用户可以在 `dev` Stage 点击发布并选择可发布 Freight。
- [ ] 用户可以将 Freight 晋级到 `test`。
- [ ] 用户可以将 Freight 晋级到 `staging`。
- [ ] 用户发起 `prod` 晋级后进入审批流程。
- [ ] `prod` 审批通过后才允许发布到生产 Stage。
- [ ] `prod` 审批禁止发起人自审批。
- [ ] PaaS 通过修改 GitLab 部署清单仓库触发 Argo CD 部署。
- [ ] 控制面不直接调用 Kubernetes API Server。
- [ ] PaaS Agent 能上报 Argo CD Application 状态。
- [ ] PaaS Agent 能上报 Workload、Pod 和 Event 状态。
- [ ] 用户能在 PaaS 中看到 Stage 状态、实例状态、发布结果和失败原因。
- [ ] 用户可以基于历史 Freight 发起回滚 Promotion。
- [ ] 回滚同样通过修改 Git 中的期望状态完成。

## 3. 失败场景验收

- [ ] 源码仓库创建失败时，PaaS 显示失败原因并记录审计日志。
- [ ] 源码仓库迁移失败时，迁移任务进入失败状态，并支持重试或取消。
- [ ] BuildSpec 不合法时，创建 Application 被阻止并返回中文错误提示。
- [ ] `java_springboot` 产物不是 jar 时，构建或构建结果处理失败。
- [ ] `java_tomcat` 产物不是 war 时，构建或构建结果处理失败。
- [ ] Jenkins 创建 Job、触发构建或读取日志失败时，PaaS 显示失败原因。
- [ ] 构建失败、取消或 unstable 时，不生成可发布 Release。
- [ ] 创建 Freight 缺少任一启用 Workload 时被阻止。
- [ ] 创建 Freight 对同一 Workload 选择多个镜像版本时被阻止。
- [ ] 创建 Freight 包含禁用或删除 Workload 时被阻止。
- [ ] Stage 处于 `pending_cluster_binding` 时，Promotion 被阻止。
- [ ] GitLab 部署清单提交失败时，Promotion 或 Deployment 进入失败状态。
- [ ] `staging` 或 `prod` 需要 MR 时，MR 创建失败会阻止后续部署。
- [ ] Argo CD Degraded 时，PaaS 能展示部署失败状态和原因。
- [ ] Pod 出现 ImagePullBackOff 时，PaaS 能展示实例失败状态和事件。
- [ ] PaaS Agent 离线时，集群状态变为离线或异常。
- [ ] 生产审批被拒绝时，Promotion 进入拒绝状态，不修改生产 Stage 期望状态。

## 4. 审计与安全验收

- [ ] 本地用户创建、密码重置、登录和 Token 签发记录审计日志。
- [ ] OIDC 登录、Identity 绑定和 OIDC Provider 变更记录审计日志。
- [ ] 租户、项目、成员和 RBAC 变更记录审计日志。
- [ ] 源码地址、构建引用、SVN revision 和 BuildSpec 变更记录审计日志。
- [ ] Application 创建、BuildSpec 固化和环境配置变更记录审计日志。
- [ ] 构建触发、取消、成功和失败记录审计日志。
- [ ] Workload 创建、部署配置变更、Freight 创建、Promotion 创建、审批、拒绝、中止和回滚记录审计日志。
- [ ] 部署模板变更、ManifestRevision 创建和 Deployment 创建记录审计日志。
- [ ] Agent token 轮换、心跳、任务下发和任务结果记录关键审计或运行日志。
- [ ] API 响应不返回 password_hash、token_hash、client_secret、kubeconfig 或外部系统凭据。
- [ ] 构建日志、审计日志和错误响应会脱敏 token、password、secret 和 registry password。
- [ ] Web Console 不提供跳转 Jenkins UI、Argo CD UI、部署清单仓库或 kubeconfig 的入口。

## 5. 前端验收

- [ ] Web Console 所有用户可见文案使用中文。
- [ ] 登录页同时提供本地账号密码登录和 OIDC 登录入口。
- [ ] 项目、应用、创建应用向导、应用详情、应用 Workload、构建日志、发布晋级、审批、回滚和审计页面可正常访问。
- [ ] 创建应用向导展示仓库、源码路径、BuildSpec 和校验结果。
- [ ] 创建 Workload 和部署配置使用弹窗或抽屉完成，不作为独立 tab。
- [ ] 发布晋级页展示 Freight 时间轴和 Stage 卡片。
- [ ] Stage 卡片点击 `发布` 后，仅当前 Stage 可发布 Freight 点亮。
- [ ] 创建 Freight 使用弹窗或抽屉完成，并强制为所有启用 Workload 选择镜像版本。
- [ ] 构建日志页面支持持续读取增量日志。
- [ ] 发布晋级页面能展示 Freight、目标 Stage、审批状态和失败原因。
- [ ] 审计页面能按关键条件查询审计日志。
- [ ] 设置真实 API 地址后，前端不依赖 mock API 完成主链路操作。

## 6. 通过标准

- [ ] 本清单中主链路、失败场景、审计与安全、前端验收均通过。
- [ ] `go test ./... -coverprofile=coverage.out` 通过。
- [ ] `web/console` 执行 `npm test -- --coverage` 通过。
- [ ] `web/console` 执行 `npm run e2e` 通过。
- [ ] `web/console` 执行 `npm run build` 通过。
- [ ] 真实 GitLab、Jenkins、Argo CD、Kubernetes 和 PaaS Agent 集成链路完成验收记录。
- [ ] 至少一个 Java Spring Boot 应用完成试点发布和回滚。
- [ ] 至少一个 Java Tomcat 应用完成试点发布和回滚。
