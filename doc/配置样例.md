# paas-server 和 paas-agent 配置样例

本文档说明测试可用版 `paas-server` 和 `paas-agent` 的启动配置。配置样例位于：

- `deploy/config/paas-server.env.example`
- `deploy/config/paas-agent.env.example`

样例文件只能保存占位值，不应写入真实密码、token、secret、kubeconfig 或生产地址。

## paas-server

当前 `paas-server` 通过环境变量读取启动配置：

```bash
set -a
source deploy/config/paas-server.env.example
set +a
go run ./cmd/paas-server
```

配置项：

| 环境变量 | 默认值 | 说明 |
| --- | --- | --- |
| `PAAS_HTTP_ADDR` | `:8080` | 控制面 HTTP 监听地址。 |
| `PAAS_AUTO_MIGRATE` | `false` | 设置为 `true` 时，启动前执行 MySQL 迁移。 |
| `PAAS_REPOSITORY_DRIVER` | `mysql` | 兼容旧部署脚本的保留变量。业务仓储固定使用 MySQL 正式表，不支持内存仓储。 |
| `MYSQL_HOST` | `127.0.0.1` | MySQL 地址。 |
| `MYSQL_PORT` | `3306` | MySQL 端口。 |
| `MYSQL_DATABASE` | `paas` | MySQL 数据库名。 |
| `MYSQL_USER` | `paas` | MySQL 用户名。 |
| `MYSQL_PASSWORD` | 空 | MySQL 密码，必须通过环境凭据注入。 |
| `GITLAB_BASE_URL` | 空 | GitLab 地址。未配置时使用 fake adapter。 |
| `GITLAB_TOKEN` | 空 | GitLab 访问 token，必须通过环境凭据注入，不得写入仓库。 |
| `GITLAB_ROOT_GROUP_PATH` | `paas` | PaaS 托管源码仓库所在 GitLab 根 Group path。 |
| `GITLAB_WEBHOOK_CALLBACK_URL` | 空 | GitLab Webhook 回调 PaaS 控制面的地址。留空时跳过 Webhook 配置；接真实 GitLab 时必须填写 GitLab 可访问的地址，不能使用 `127.0.0.1`。 |
| `GITLAB_TIMEOUT_SECONDS` | `10` | GitLab HTTP 调用超时时间。 |
| `GITLAB_RETRY_MAX` | `2` | GitLab 临时失败重试次数。 |
| `JENKINS_BASE_URL` | 空 | Jenkins 地址。未配置时使用 fake adapter。 |
| `JENKINS_USERNAME` | 空 | Jenkins API 用户名。 |
| `JENKINS_TOKEN` | 空 | Jenkins API token，必须通过环境凭据注入，不得写入仓库。 |
| `JENKINS_TIMEOUT_SECONDS` | `10` | Jenkins HTTP 调用超时时间。 |
| `JENKINS_RETRY_MAX` | `2` | Jenkins 临时失败重试次数。 |
| `JENKINS_DEFAULT_TEMPLATE_ID` | `java-unified-v1` | 兼容历史 Jenkins 模板的默认 ID，新构建流程优先使用全局构建模板。 |
| `JENKINS_CALLBACK_BASE_URL` | `http://127.0.0.1:8080` | Jenkins 回调 PaaS 控制面的基础地址，构建时会拼接 `/api/builds/{buildRunId}/callback`。 |
| `IMAGE_REPOSITORY` | `registry.local/order-api` | 构建产物镜像仓库前缀，触发构建时追加应用名称。 |
| `DOCKERFILE_REPOSITORY_URL` | `ssh://git@192.168.100.80:2422/paas/dockerfiles.git` | 平台统一 Dockerfile 仓库地址。 |
| `DOCKERFILE_REPOSITORY_REF` | `main` | 平台 Dockerfile 仓库 ref，建议使用 tag 或 commit。 |
| `DOCKERFILE_REPOSITORY_CREDENTIALS_ID` | 空 | Jenkins 中用于检出平台 Dockerfile 仓库的凭据 ID。 |

`PAAS_AUTO_MIGRATE=true` 时，启动流程会读取 `MYSQL_*`，连接 MySQL 并执行 `internal/migrations.All()` 中的迁移。迁移失败时服务不会继续启动。

`paas-server` 使用 MySQL 正式表持久化业务仓储。运行期不再提供内存仓储，也不再使用 `repository_snapshots` 快照表；迁移 `drop_repository_snapshots` 会删除历史快照表，旧快照数据不迁移。

配置 `GITLAB_BASE_URL` 和 `GITLAB_TOKEN` 后，SourceRepository 创建会调用真实 GitLab，并按 `GITLAB_ROOT_GROUP_PATH/Tenant/Project/SourceRepository` 确保 Group/Subgroup/Project。未配置 GitLab 时继续使用 fake adapter，便于本地开发。

配置 `JENKINS_BASE_URL` 和 `JENKINS_TOKEN` 后，构建模块会调用真实 Jenkins：平台管理员先在 Web Console 的“构建管理”页面维护构建环境、运行时环境和全局构建模板，用户创建 BuildPipeline 时选择已启用构建环境和一个或多个运行时环境。触发构建时 PaaS 会按 `paas/{tenant_name}/{project_name}/{app_name}` 确保 Jenkins Folder/Job，Job 不存在时创建，已存在时按本次 BuildRun 重新渲染 Jenkinsfile 后更新，触发 Jenkins 时不传递构建参数，并通过 progressiveText 拉取日志。Jenkins 会增量检出应用源码和平台 Dockerfile 仓库，Spring Boot jar 使用 `java/jar/Dockerfile`，Tomcat war 使用 `java/tomcat/Dockerfile`，并通过 `docker buildx` 构建多架构镜像。Jenkins 构建节点应提供持久化 `/backup_data` 目录，用于 Maven、Gradle、npm、Yarn、pnpm 依赖缓存和 buildx layer cache；这些缓存默认是节点本地缓存，跨节点调度时不会自动共享。未配置 Jenkins 时继续使用 fake adapter。

## paas-agent

当前 `paas-agent` 通过环境变量读取启动配置：

```bash
set -a
source deploy/config/paas-agent.env.example
set +a
go run ./cmd/paas-agent
```

配置项：

| 环境变量 | 默认值 | 说明 |
| --- | --- | --- |
| `PAAS_CLUSTER_ID` | 无 | Agent 所属集群 ID，必须与控制面 Cluster 记录一致。 |
| `PAAS_CONTROL_PLANE_URL` | 无 | PaaS 控制面地址，Agent 调用 `/api/agent/v1/*`。 |
| `PAAS_AGENT_TOKEN` | 无 | 控制面生成的 Agent token，应通过 Kubernetes Secret 注入。 |
| `PAAS_AGENT_NAMESPACES` | 空 | 逗号分隔的采集 namespace，例如 `argocd,order-dev,order-test`。 |
| `PAAS_HEARTBEAT_INTERVAL` | `10s` | Agent 心跳周期。 |
| `PAAS_SNAPSHOT_INTERVAL` | `30s` | 状态快照上报周期。 |

Agent 请求控制面时会携带：

- `Authorization: Bearer <PAAS_AGENT_TOKEN>`
- `X-PaaS-Cluster-ID: <PAAS_CLUSTER_ID>`

控制面必须校验 token 只能上报所属集群，避免跨集群冒用。

## Kubernetes Secret 注入示例

`PAAS_AGENT_TOKEN` 不应出现在 ConfigMap 或普通 manifest 中。示例：

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: paas-agent-secret
  namespace: paas-system
type: Opaque
stringData:
  PAAS_AGENT_TOKEN: "replace-with-control-plane-issued-token"
```

Deployment 中通过 `envFrom` 或单项 `secretKeyRef` 注入该 Secret。RBAC 草案见 `deploy/paas-agent/rbac.yaml`。

## 安全约束

- 不在代码、文档或日志中记录真实 token、密码、secret、kubeconfig。
- `.env` 实际配置文件应只保存在部署环境，并设置最小读取权限。
- Web Console 是用户入口；普通用户不直接访问 Jenkins、Argo CD、部署清单仓库或 kubeconfig。
- PaaS 控制面不直接调用 Kubernetes API Server，集群侧状态和受控任务由 `paas-agent` 完成。
