# 健康检查、配置校验、关键日志和基础指标

本文档记录测试可用版最小可观测能力。

## paas-server

`paas-server` 提供以下检查接口：

| 接口 | 用途 | 成功响应 |
| --- | --- | --- |
| `GET /healthz` | 进程存活检查 | `{"status":"ok"}` |
| `GET /readyz` | 控制面依赖就绪检查 | `{"status":"ready", ...}` |
| `GET /metrics` | Prometheus 文本格式基础指标 | `text/plain; version=0.0.4` |

验证命令：

```bash
curl -fsS http://127.0.0.1:8080/healthz
curl -fsS http://127.0.0.1:8080/readyz
curl -fsS http://127.0.0.1:8080/metrics
```

当前 `/metrics` 暴露以下 gauge：

- `paas_ready`
- `paas_projects_total`
- `paas_applications_total`
- `paas_environment_states`
- `paas_build_runs_total`
- `paas_freights_total`
- `paas_audit_logs_total`

这些指标用于测试可用版试点的基础观测，不包含用户 token、外部系统凭据、kubeconfig 或 secret 内容。

关键启动日志：

- MySQL 迁移失败时，`paas-server` 停止启动并输出 `执行 MySQL 迁移失败`。
- 控制面依赖初始化失败时，输出 `初始化 PaaS 控制面失败`。
- HTTP 服务启动后，输出 `PaaS 控制面已启动` 和监听地址。

## paas-agent

`paas-agent` 启动时会校验以下必填配置：

- `PAAS_CLUSTER_ID`
- `PAAS_CONTROL_PLANE_URL`
- `PAAS_AGENT_TOKEN`

缺少必填项时，进程停止启动并输出 `paas-agent 配置校验失败`。校验通过后输出集群 ID、心跳周期、快照周期和 Argo CD namespace，不输出 Agent token。随后 Agent 会使用集群内 ServiceAccount 访问 Kubernetes API，向控制面发送心跳、状态快照和任务结果。

部署前渲染验证：

```bash
helm lint deploy/paas-agent
helm template paas-agent deploy/paas-agent --namespace paas-system \
  --set config.clusterID=cluster_dev \
  --set config.controlPlaneURL=http://paas-server.paas-system.svc.cluster.local:8080 \
  --set secret.token=change-me
```

预期输出示例：

```text
paas-agent cluster=cluster_dev heartbeat=10s snapshot=30s argocd_namespace=argocd
```

Kubernetes 部署验证：

```bash
kubectl -n paas-system logs deployment/paas-agent
```

## 后续增强

- 测试环境接入 Prometheus 后，应按命名空间和服务名抓取 `/metrics`。
- 真实集成阶段应增加 GitLab、Jenkins、Argo CD、Registry 调用成功率、失败次数和耗时指标。
- Agent 后续提供本地健康端口后，应在 Helm Chart 中添加 Kubernetes liveness/readiness probe。
