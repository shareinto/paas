# 凭据、token、secret 脱敏和暴露面检查

本文档记录测试可用版试点前的敏感信息暴露面检查结果。检查范围包括文档、后端、Agent、前端源码和部署样例，不包含 `node_modules`、`dist`、coverage 产物和 `coverage.out`。

## 检查命令

```bash
rg -n "(?i)(password|passwd|pwd|token|secret|kubeconfig|client_secret|private-token|authorization|bearer|密码|凭据|密钥)" . \
  --glob '!web/console/node_modules/**' \
  --glob '!web/console/dist/**' \
  --glob '!web/console/coverage/**' \
  --glob '!coverage.out'
```

## 已修复问题

- `doc/中间件部署.md` 曾记录开发环境服务器登录密码，已移除。该文档现在只保留服务器地址、运行目录和凭据文件路径，不记录任何实际密码。

## 当前结论

- 配置样例只包含 `change-me`、`replace-with-control-plane-issued-token` 等占位值。
- `deploy/config/paas-server.env.example` 不包含真实 MySQL 密码。
- `deploy/config/paas-agent.env.example` 不包含真实 Agent token。
- `doc/配置样例.md` 和 `doc/本地和测试环境部署.md` 明确要求真实凭据通过环境或 Kubernetes Secret 注入。
- `identity-access` 模块使用 bcrypt 保存本地密码哈希，AccessToken/RefreshToken 只保存哈希。
- OIDC Provider DTO 不返回 `client_secret_ref` 或 client secret。
- `cluster-agent` 模块只在注册和轮换时返回一次性 Agent token，列表和查询响应隐藏 `agent_token_hash`。
- `build` 和 Jenkins 集成日志会脱敏 `token`、`password`、`secret`、`PAAS_TOKEN`、`GITLAB_TOKEN`、`REGISTRY_PASSWORD`。
- Web Console 不提供 Jenkins UI、Argo CD UI、部署清单仓库或 kubeconfig 跳转入口。

## 允许存在的非真实敏感关键词

以下内容属于测试、占位、字段名或安全约束说明，不构成真实凭据泄露：

- 测试中的 `password`、`secret`、`token` 字符串。
- API 字段名，例如 `password`、`token`、`agent_token`、`client_secret_ref`。
- 文档中的安全约束、验收项和环境变量名。
- 前端 localStorage key `paas_token` 和 mock token。

## 后续要求

- 不提交实际 `.env`、kubeconfig、私钥、证书私钥或生产 token。
- 新增 GitLab、Jenkins、Registry、OIDC、Argo CD 配置读取时，必须使用 Secret 引用或环境注入，并补充脱敏测试。
- 真实集成验收前，需要在部署环境再次执行敏感关键词扫描，并人工复核命中项。
