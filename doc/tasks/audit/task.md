# audit 测试可用版本任务

## 目标

实现统一审计日志能力，为所有关键操作提供不可变审计记录和查询接口。

## 任务清单

- [x] 定义领域模型：`AuditLog`。
- [x] 建立数据表迁移草案：`audit_logs`。
- [x] 定义审计字段：操作者、主体类型、租户、项目、资源类型、资源 ID、动作、结果、摘要、时间。
- [x] 实现 `AuditLogger` port，供其他模块写审计日志。
- [x] 实现审计日志不可变写入，禁止更新和删除。
- [x] 实现按租户、项目、用户、资源、动作、时间范围查询。
- [x] 覆盖关键事件：登录、创建租户、创建项目、修改成员权限、创建应用、迁移仓库、触发构建、取消构建、创建 Freight、创建 Promotion、审批或拒绝发布、修改环境配置、修改密钥、回滚、禁用集群、Agent 离线。
- [x] 覆盖 BuildSpec 变更审计：build_command、artifact_copy_command、runtime_base_image、artifact_deploy_path。
- [x] 提供 API：审计日志列表、审计日志详情。
- [x] 编写测试：不可变写入、字段完整性、条件查询、敏感字段不落库。

## 完成标准

- [x] 所有关键操作可审计。
- [x] 审计日志不可变。
- [x] Secret、token、password 不进入审计明文。
- [x] 测试通过。
